Cubic

Background

Cubic forms were known by many ancient groups of people, including the Babylonians, Greeks, Chinese, Indians, and Egyptians. Babylonian cuneiform tablets from around 2000 BC to 1500 BC with calculations for cubes and cube roots. The problem of doubling the cube involves a very old cubic function, and today, it is known to be impossible. Doubling the cube involves using an unmarked straightedge and a compass in order to construct a cube that has double the volume of another cube. This cannot be done without other tools. In The Nine Chapters on the Mathematical Art, methods for solving cubic equations were compiled. This is a Chinese math book that was written some time between 10th and 2nd century BC. During the 3rd century, the Greek mathematician Diophantus found integer or rational solutions for some bivariate cubic equations. His work later led to the creation of what are now called Diophantine equations. Much later in history, during the 11th century a Persian poet-mathematician named Omar Khayyám further advanced the world’s knowledge of cubic equations. He discovered that cubic functions are able to have multiple solutions and that these are not able to be found with just a compass and straightedge. Later on, he wrote a complete classification of cubic equations and some solutions there were found through the use of intersecting conic sections. Over the course of the next 500 years, more and more was found out about these functions and how to effectively solve them. Today, cubic functions are used most commonly as Bézier curves. The curves present in in many drawing and graphics programs are made out of these Bézier curves. Almost all of the curves that are seen in video games are these kind of curves, too, and so are many fonts like Georgia and Times New Roman. Also, Kepler’s third law of planetary motions has tons of squares and cubes involved. The parent function is y = x3


How to Graph

Graphing the cubic parent function is very simple if you are able to perform simple multiplication on paper or in your head. Think of any number, that number will become your “x” value for a certain point. Next, multiply that “x” value times itself, and then multiply that product by itself again. That’ll be your “y” value for that point. On the parent function, the origin itself is a point because anything multiplied by 0 is 0, so multiplying it by itself using the previously mentioned rule wouldn’t do anything. Next up, you have 1. If you multiply 1 by itself, you get 1, no matter what. So another point you can add is (1,1). The next “x” value would be 2, which would create a “y” value of 8 (2 times 2 is 4, and then multiplying that by 2 gets you 8). Using this method, you can go as far as you want with the “x” and “y” values because the domain and range are both negative infinity to positive infinity. Interestingly enough, adding points with a negative “x” creates a negative “y”. This is because, when you multiply a negative number by itself, the negative cancels out. But because you have to multiply the “x” value by itself twice, the negative comes back (a negative number multiplied by a positive number creates a negative product). The part of the function in the first quadrant gets rotated 180 degrees to become the portion in the third quadrant. The “y” values also get very big very quickly because of the multiplying involved. In accordance to other functions, the general form is very simple, and allows the graph to be changed in many ways. In the equation y=a(b(x - h))3 + k, the “a” value dictates vertical growth, the “b” value dictates horizontal growth, and the “h” and “k” value are the starting point of the equation in the form (h, k). Changing the “a” or “b” value results in a vertical or horizontal stretch / compression, and changing the “h” and / or “k” value results in a translation. Whenever you do a vertical stretch or compression, the “k” value is also affected, and whenever you do a horizontal stretch or compression, the “h” value is affected, too. For example, if you have the function y=5(x + 3)3-7, and you wanted to give it a vertical compression by 1⁄2, you would multiply the entirety of the function by 1⁄2, leaving the function as y=5⁄2(x + 3)3 - 7⁄2. Also, if you want to do a horizontal stretch or compression, you have to input the reciprocal of the stretch or compression you want to do. If you were to do a horizontal stretch by 3, you would input 1⁄3 as the “b”. y = (x - 4)3. would turn into y = (1⁄3(x - 4))3, which simplifies to y=(x⁄3 - 4⁄3)3. As a side note, whenever the “a” or “b” value is 1, it is not visible in the function as it has no effect (these values are always multiplied). If the “a” value is negative, a reflection over the x-axis is created, and if the “b” value of y=a(b(x - h))3 + k is negative, the function is reflected over the y-axis. The equation can also be multiplied out and written as y=ax3+bx2+cx+d but this is much harder to work with. This form only allows you to figure out the y-intercept. If you want to know things like the solutions, you have to use the form mentioned above. However, your equation will only look exactly like that if there is only one solution. More often, you’ll see something like y=2(x+1)(x+4)(x-5)-3, and it functions slightly differently. All of the binomials are used to find the x-intercepts instead of the vertex. The value at the very front dictates whether you have a vertical stretch or compression, just as before.

Transformations

y = 1⁄5(-6x + 6)3 + 5
First, we have to factor out -6 from -6x+6 in order to find the “b” value and find the actual horizontal translation.
y = 1⁄5(-6(x - 1))3 + 5
The 1⁄5 at the beginning signifies that that there is a vertical compression by a factor of 1⁄5. All of the y-values on the function’s points are divided by 5.
The -6 “b” value signifies that there is a horizontal compression by a factor of 1⁄6 (horizontal stretches and compressions are reciprocals). All of the “x” value on the function’s points are divided by 6. The negative “b” value also indicates that this function is reflected over the y-axis.
Next, the “h” value in the function looks like -1, but in actuality, it’s 1 because there is a subtraction symbol in the general form. This indicates that after the previous translations, the function will be moved to the right by 1 unit.
Finally, the “k” value in this equation is 5, meaning the entire function will be translated upwards by 5 units.
